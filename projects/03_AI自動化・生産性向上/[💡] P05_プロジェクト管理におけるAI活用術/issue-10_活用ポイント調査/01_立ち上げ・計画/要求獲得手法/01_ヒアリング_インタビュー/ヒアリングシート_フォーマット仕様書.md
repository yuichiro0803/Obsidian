# 業務ヒアリングシート　フォーマット仕様書
> 他のAIがこのドキュメントを忠実に再現するための設計書

---

## 0. このドキュメントの目的と概要

本書は「業務ヒアリングシート（生成AIサポート版）」の構造・デザイン・データ定義を、他の生成AIが正確に再現できるよう仕様化したものです。Node.js の `docx` ライブラリ（v9系）を用いた実装を前提としています。

> **前提：** `npm install -g docx` でインストール済みであること。テーブルの幅は DXA 単位（1440 DXA = 1インチ）で指定。`ShadingType` は必ず `CLEAR` を使用（`SOLID` は背景が黒くなる）。

---

## 1. ページレイアウト・基本設定

| 項目 | 設定値 | 詳細・注意 |
|---|---|---|
| 用紙サイズ | A4 縦 | 11906 × 16838 DXA |
| 余白（上下左右） | 各 2cm | 1134 DXA（≒ 2cm） |
| フォント | Arial 統一 | 本文 10pt（size:20）・見出し 12pt（size:24）・小 9pt（size:18） |
| ヘッダー | タイトル＋右揃えプロジェクト名 | 下線：紫（#534AB7）・太さ 6 / タブストップ右端 9026 |
| フッター | 社外秘ラベル＋右揃えページ番号 | 上線：グレー（#D3D1C7）・太さ 4 / タブストップ右端 9026 |
| コンテンツ幅 | 9026 DXA | A4幅 11906 − 左右余白 2×1134 = 9026 DXA。全テーブルはこの幅に揃える |

```javascript
// ページ設定（A4縦・余白2cm）
properties: {
  page: {
    size: { width: 11906, height: 16838 },
    margin: { top: 1134, right: 1134, bottom: 1134, left: 1134 },
  },
},
```

## 3. ドキュメントのセクション構成

ドキュメントは以下の7セクションで構成されます。各セクションはセクション見出し（下線付き・紫）で区切ります。

| 順序 | セクション名 | 要素 | 内容・備考 |
|:---:|---|---|---|
| ① | タイトル | `Paragraph` | 48pt太字・紫。副題を20pt・グレーで続ける |
| ② | 基本情報 | `Table` 4列×2行 | 8フィールド（日付・担当者・記録者・PJ名・業務・部門・役職・時間）。ラベル太字・背景 `grayLight` |
| ③ | 使い方ガイド | `Paragraph` ×4 | 番号付き4箇条。最後の1行のみ文字色 `coral`（警告） |
| ④ | 質問票（メイン） | `Table`（可変行数） | 5列テーブル。カテゴリ行・質問行・回答行の3種類の行を繰り返す（詳細は §4） |
| ⑤ | 特記事項 | `Table` 1列 | 自由記述欄。高さ確保のため空行×5を含む |
| ⑥ | 次回アクション | `Table` 4列×4行 | No.・アクション・担当・期日。空行で記入可能に |
| ⑦ | AI活用ガイド | `Table` 2列×5行 | 活用場面とAIへの指示内容の早見表 |

## 4. 質問票テーブルの詳細仕様

### 4-1. 列定義

| 列名 | 幅（DXA） | 書式 | 内容・注意点 |
|---|---|---|---|
| No. | 500 | 中央揃え・太字 | 通し番号（1〜）。カテゴリ行・回答行には不要 |
| カテゴリ | 1300 | 左揃え・通常 | 現状把握 / 課題 / 例外 / 要望 / 制約 など |
| 質問内容 | 3660 | 左揃え・通常 | 実際にヒアリングで読み上げる質問文。フォント14pt相当 |
| 確認の目的 | 2200 | 左揃え・グレー文字（`textSub`） | なぜこの質問が必要か。記録者・レビュアー向けメモ |
| 要確認（確信度） | 700 | 中央揃え・色付き | 高=緑背景 / 中=黄背景 / 低=赤背景。仮説の薄さを示す |

> **注意：** 列幅の合計（500+1300+3660+2200+700 = **8360**）は `columnWidths` 配列の合計と `width.size` が一致している必要があります。コンテンツ幅 9026 に合わせる場合は各列幅を比例調整してください。

### 4-2. 行の種類と書式

| 行種別 | 列の扱い | 背景色 | 枠線色 | テキスト | 備考 |
|---|---|---|---|---|---|
| **カテゴリ行** | 5列結合（`columnSpan: 5`） | `EEEDFE` | `AFA9EC` | 太字・紫文字（`534AB7`） | カテゴリ番号と名称を記載。例：「1. 現状業務の流れ（AS-IS）」 |
| **質問行** | 5列個別セル | `FFFFFF` / `FAFAFA`（交互） | `D3D1C7` | 通常 | 奇数行白・偶数行 `FAFAFA` |
| **回答行** | 5列結合（`columnSpan: 5`） | `FAFAFA` | `D3D1C7` | グレー文字・斜体（`textHint`） | 「【回答記入欄】」と薄く表示。高さ確保のため空行2〜3行追加 |

```javascript
// カテゴリ行の実装例（5列結合）
new TableRow({ children: [
  new TableCell({
    columnSpan: 5,
    shading: { fill: "EEEDFE", type: ShadingType.CLEAR },
    borders: allBorders("AFA9EC"),
    children: [ new Paragraph({
      children: [ new TextRun({ text: "1. 現状業務の流れ（AS-IS）",
        bold: true, color: "534AB7", font: "Arial", size: 20 }) ]
    }) ]
  })
] })
```

### 4-3. 確信度（要確認）の色定義

質問票の最右列「要確認」は仮説の確信度を視覚化します。

| 値 | 背景色（fill） | 文字色 | 枠線色 | 意味・ヒアリング上の扱い |
|:---:|---|---|---|---|
| **高** | `E1F5EE` | `085041` | `0F6E56` | 資料・規程に明記されており仮説として確実。ヒアリングで軽く確認する程度でよい |
| **中** | `FAEEDA` | `854F0B` | `BA7517` | 推測できるが未確認。ヒアリングで詳細条件を確認すべき |
| **低** | `FAECE7` | `993C1D` | `D85A30` | 資料から読み取れない。ヒアリングで必ず確認が必要な最重要項目 |

### 4-4. 質問データの構造

質問データは以下の形式の配列として定義し、ループでテーブル行を生成します。

```javascript
// 質問データ配列の構造
const questions = [
  // カテゴリ区切り行
  { type: "cat", label: "1. 現状業務の流れ（AS-IS）" },

  // 質問行
  { type: "q",
    no: 1,
    cat: "現状把握",
    q: "受注から出荷までの業務フローを教えてください",
    p: "全体フローの把握・担当者の特定",
    c: "低"  // 確信度: 高 / 中 / 低
  },

  // 回答記入行
  { type: "ans", note: "" },  // note があれば補足として表示

  // 次のカテゴリへ...
];
```

## 6. ヘッダー・フッターの仕様

### 6-1. ヘッダー（Header）

全ページ共通。タイトル文字列を左に、プロジェクト名を右に配置します。タブストップで右揃えを実現します。

```javascript
// ヘッダー実装パターン
new Header({ children: [
  new Paragraph({
    spacing: { after: 0 },
    border: { bottom: { style: BorderStyle.SINGLE, size: 6, color: "534AB7", space: 4 } },
    tabStops: [{ type: TabStopType.RIGHT, position: 9026 }],
    children: [
      new TextRun({ text: "業務ヒアリングシート　─　生成AIサポート版",
        bold: true, font: "Arial", size: 18, color: "534AB7" }),
      new TextRun({ text: "\t" }),   // タブで右端へ
      new TextRun({ text: "プロジェクト名", font: "Arial", size: 16, color: "B4B2A9" }),
    ],
  }),
] })
```

### 6-2. フッター（Footer）

全ページ共通。左にラベル文字列、右にページ番号を配置します。`PageNumber.CURRENT` を使用します。

```javascript
// フッター実装パターン
new Footer({ children: [
  new Paragraph({
    border: { top: { style: BorderStyle.SINGLE, size: 4, color: "D3D1C7", space: 4 } },
    tabStops: [{ type: TabStopType.RIGHT, position: 9026 }],
    children: [
      new TextRun({ text: "社外秘　─　本シートは要件定義プロジェクト用です",
        font: "Arial", size: 16, color: "B4B2A9" }),
      new TextRun({ text: "\t" }),
      new TextRun({ children: [PageNumber.CURRENT], font: "Arial", size: 16 }),
    ],
  }),
] })
```

## 7. 実装上の重要な注意点

> ⚠️ 以下のルールを守らないとテーブル崩れ・黒背景・フォント不一致などが発生します。必ず守ってください。

### 1. テーブル幅は必ず DXA で指定
`WidthType.PERCENTAGE` は Google Docs で崩れます。常に `WidthType.DXA` を使用してください。テーブルの `width.size` と `columnWidths` の合計が一致しなければなりません。

### 2. ShadingType は CLEAR を使用
`ShadingType.SOLID` を指定すると背景が強制的に黒になります。常に `ShadingType.CLEAR` を使用してください。

### 3. セル幅はテーブルと両方設定
`columnWidths`（テーブル側）と各セルの `width`（セル側）を両方設定しないと、一部プラットフォームで列幅がずれます。

### 4. 改行は Paragraph を分ける
`TextRun` 内に `\n` を使ってはいけません。新しい行が必要な場合は別の `Paragraph` 要素を追加してください。

### 5. 箇条書きは LevelFormat を使用
`TextRun` に「•」などのユニコード文字を直接入れてはいけません。`numbering config` と `LevelFormat.BULLET` / `DECIMAL` を使用してください。

### 6. セルの内側余白を設定
`margins: { top: 80, bottom: 80, left: 120, right: 120 }` を各セルに設定してください。未設定だとテキストが枠線に密着して読みにくくなります。

### 7. ヘッダー・フッターに Table を使わない
`Table` をヘッダー・フッター内に置くと最小の高さが発生して意図しない余白になります。タブストップで2列レイアウトを実現してください。

## 8. 業務・プロジェクトごとのカスタマイズポイント

以下の箇所を変更するだけで、異なる業務・プロジェクト向けのヒアリングシートを生成できます。

| 変更箇所 | 変更内容の例 |
|---|---|
| 対象業務名 | 「受注処理」→「経費精算」「採用管理」「在庫管理」など |
| `questions` 配列 | 業務に合わせた質問文・カテゴリ・確信度をAIで生成して差し替え |
| カテゴリ数・名称 | 5カテゴリ固定ではなく、業務の複雑さに応じて増減可能 |
| ヘッダー右側のテキスト | 「© システム開発プロジェクト」→ 実際のPJ名・組織名に変更 |
| フッターのラベル | 「社外秘」→「取扱注意」「部外秘」など機密分類に応じて変更 |
| アクセントカラー | `purple`（#534AB7）を組織のブランドカラーに置き換え可能 |
