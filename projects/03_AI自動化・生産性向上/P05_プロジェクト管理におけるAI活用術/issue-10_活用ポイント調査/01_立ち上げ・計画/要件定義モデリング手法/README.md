# 要件定義モデリング手法

要件定義書の作成において、業務や機能を「図・モデル」として表現するための標準的な手法をまとめています。各手法についての詳細と、AIを活用した作成・活用方法を個別ドキュメントで解説します。

## 手法一覧

| フォルダ | 手法名 | 主な用途 |
| :--- | :--- | :--- |
| [01_業務フロー図](./01_業務フロー図/) | 業務フロー図（AS-IS / TO-BE） | 現行・将来の業務の流れを可視化 |
| [02_ユースケース図](./02_ユースケース図/) | ユースケース図（UML） | システムとユーザーの相互作用を定義 |
| [03_DMM](./03_DMM/) | DMM（ドメインモデル図） | 業務領域の概念・関係を整理 |
| [04_ER図](./04_ER図/) | ER図（論理データモデル） | データの構造・関係を設計 |
| [05_状態遷移図](./05_状態遷移図/) | 状態遷移図 | データやエンティティの状態変化を定義 |
