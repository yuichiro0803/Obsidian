# 業務フロー図：手法詳細

## 概要
業務フロー図（Business Process Flow）は、業務の一連の流れをステップ・担当者・判断分岐を用いて図式化する手法です。要件定義では**現行業務（AS-IS）**と**将来業務（TO-BE）**の2種類を作成し、改善の方向性と新システムのスコープを明確にします。

## 目的と効果
- 関係者全員が業務の流れを共通認識できる
- システム化の対象範囲（自動化すべき箇所）を特定できる
- 担当者・部門間の引き継ぎポイントと責任境界を明確にできる
- AS-ISとTO-BEの比較によって、システム化による改善効果を可視化できる

## 種類と使い分け
| 種類 | 説明 | 活用シーン |
| :--- | :--- | :--- |
| **AS-IS フロー** | 現行の業務プロセスをそのまま記述 | 現状把握・ヒアリング準備・問題点の特定 |
| **TO-BE フロー** | 新システム導入後の理想の業務プロセス | スコープ定義・ユーザー合意形成 |
| **スイムレーン図** | 担当者・部門ごとに縦横にレーンを区切ったフロー | 複数部門をまたぐ業務の可視化 |

## 記法・表記ルール
| 図形 | 意味 |
| :--- | :--- |
| 角丸四角（スタート/エンド） | 業務の始まりと終わり |
| 四角 | 作業・アクション |
| ひし形 | 判断・条件分岐（Yes/No） |
| 矢印 | 処理の流れ（方向） |
| 円筒 | データベース・帳票・ファイル |
| 平行四辺形 | 入力・出力 |

## 作成ステップ
1. **スコープ定義**：フローの開始条件（トリガー）と終了条件を決める
2. **ステップの洗い出し**：ヒアリング・既存資料をもとに業務ステップを列挙
3. **担当者の整理**：各ステップの担当者（役割）を対応させる
4. **判断分岐の明記**：条件と分岐先を明確にする
5. **使用ツール・システムの追記**：各ステップで使用する道具・システムを付記
6. **レビューと合意**：業務担当者にレビューしてもらい、認識のズレを修正

## チェックリスト
- [ ] 開始条件（トリガー）と終了条件が明記されているか
- [ ] 全ての判断分岐に「Yes/No」が定義されているか
- [ ] 各ステップの担当者が特定されているか
- [ ] 例外処理・エラー時のフローが含まれているか
- [ ] 使用するツール・システムが記載されているか
- [ ] 業務担当者のレビューが完了しているか
