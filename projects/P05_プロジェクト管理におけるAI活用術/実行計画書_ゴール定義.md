# P05｜実行計画書 - プロジェクト管理におけるAI活用術

> 本ドキュメントは「このプロジェクトで何を達成するか」「いつ・誰が・何をするか」を明確にする実行計画書です。

---

## 1. なぜこのプロジェクトをやるのか（Why）

### 現状の問題

```
プロジェクト管理の各プロセスをすべて人力で回している

具体的な弊害：
  ・WBS作成・タスク分解に時間がかかる
  ・進捗管理・議事録作成などルーティン業務の工数が大きい
  ・AIを使える場面があっても、どう使えばよいか共有されていない
```

### このプロジェクトで解決すること

> **プロジェクト管理にAIを組み込み、工数削減と成果の質向上を両立する手法を確立する**

---

## 2. 最終ゴール（What）

### ゴール文（1文で言うと）

> **チーム全員が使えるAIガイドラインを策定し、プロジェクト管理の工数を削減しながら成果の質を高めること**

### ゴール状態の定義

| # | ゴール | 達成基準 |
|---|-------|---------|
| G1 | AI活用ポイントが洗い出されている | プロジェクト管理の各プロセスでAIが使える箇所が一覧化されている |
| G2 | WBS・タスク分解の自動化が検証されている | AIによるWBS生成の精度・限界が把握されている |
| G3 | チーム向けAIガイドラインが完成している | AI活用ガイドラインがoutputsフォルダに格納され、チームに共有できる状態 |
| G4 | 実際にプロジェクト管理の工数が削減されている | 体感・定量両面でのフィードバックが得られる |

---

## 3. スコープとスケジュール（Scope / When）

### 対象範囲

| 対象 | 内容 |
|------|------|
| **対象プロセス** | WBS作成・タスク分解・進捗管理・議事録・リスク管理 |
| **対象ツール** | Antigravity（AIエージェント）、GitHub / Backlog、Obsidian |
| **対象チーム** | まず自プロジェクトで検証し、知見をまとめる |

### フェーズ別スケジュール

```
Phase 1｜調査（#10）  ⬜ 未着手
  └─ AI活用ポイントの洗い出し・他社事例調査

Phase 2｜検証（#11）  ⬜ 未着手
  └─ WBS自動化・タスク分解の実験と精度確認

Phase 3｜ガイドライン策定（#12）  ⬜ 未着手
  └─ チーム向けAI活用ガイドラインの作成・共有
```

---

## 4. 実行計画（Who / Commitment）

| コミットメント | 具体的な行動 |
|-------------|------------|
| 調査を完了させる | AI活用できるPM業務の候補を10個以上洗い出す |
| 検証を実施する | 実際のプロジェクト業務でAIを使い、効果と課題を記録する |
| ガイドラインを完成させる | チームが明日から使えるレベルのガイドをまとめる |

---

## 5. 成功指標（KPI）

| 指標 | 目標値 |
|------|-------|
| AI活用ポイントの洗い出し数 | 10項目以上 |
| 検証した自動化パターン数 | 3パターン以上 |
| ガイドラインの完成 | チームへの共有が完了する |

## 6. ゴール別タスク・進捗管理（How）

### 🟢 G1-G2: AI活用ポイントの洗い出しと検証
- [x] プロジェクト管理におけるAI活用箇所のリストアップ（#10）
- [ ] AIによるWBS・タスク分解の精度検証（#11）
- [ ] ルーティン業務（議事録・進捗報告等）へのAI適用実験

- [ ] 選定した業務プロセスでのAI本格運用
- [ ] 導入前後での工数・品質の変化の測定
- [ ] 改善フィードバックループの構築

## 7. 迷ったときの判断基準

```
Q：どの施策から始めるべきか？
  → #10（調査）から。調査なしで検証・ガイドを書くと質が落ちる

Q：AIを活用しようとして効果が出なかったとき
  → 「#11 検証」イシューに記録を残す。失敗も知見

Q：ガイドラインの粒度が分からない
  → 「チームメンバーが明日から使えるか？」を判断基準にする
```
