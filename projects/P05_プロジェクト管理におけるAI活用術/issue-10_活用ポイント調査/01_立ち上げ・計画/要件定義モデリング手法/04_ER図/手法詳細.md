# ER図（論理データモデル）：手法詳細

## 概要
ER図（Entity-Relationship Diagram）は、システムが扱うデータの構造・エンティティ・属性・エンティティ間の関係をモデル化した図です。要件定義では**論理ER図**（実装に依存しないデータの構造定義）を作成し、DB設計の土台を確立します。

## 目的と効果
- システムが保存・管理すべきデータを漏れなく定義できる
- データの重複・矛盾・冗長性を設計段階で排除できる
- データ移行（既存システムから新システムへの移行）の設計に活用できる
- DB設計者・開発者との認識合わせの根拠資料になる

## ER図の種類
| 種類 | 説明 | 作成タイミング |
| :--- | :--- | :--- |
| **概念ER図** | 主要なエンティティと関連のみ。属性は省略 | 要件定義初期（DMMに近い） |
| **論理ER図** | 全エンティティ・属性・PK・関係の多重度を定義 | 要件定義後半〜基本設計 |
| **物理ER図** | テーブル名・カラム名・データ型・インデックスを含む | 詳細設計 |

## 主要な要素
| 要素 | 記号 | 意味 |
| :--- | :--- | :--- |
| **エンティティ** | 四角（上段：エンティティ名、下段：属性） | データの主役となる概念 |
| **主キー（PK）** | 下線付き * | そのエンティティを一意に識別するキー |
| **外部キー（FK）** | #付き | 他のエンティティを参照するキー |
| **関連（リレーション）** | 線 | エンティティ間のデータ的な関係 |
| **多重度（カーディナリティ）** | 1, *, 0..1 | 1対1、1対多、多対多などの数量関係 |

## カーディナリティ（多重度）の記法
| 記法 | 意味 |
| :--- | :--- |
| `1 --- 1` | 1対1（1件に対して必ず1件） |
| `1 --- *` | 1対多（1件に対して0件以上） |
| `1 --- 1..*` | 1対1以上（1件に対して1件以上は必須） |
| `* --- *` | 多対多（中間テーブルで解消する） |

## 作成ステップ
1. **DMMを参照**：ドメインモデル図から主要エンティティを引き継ぐ
2. **属性の洗い出し**：各エンティティが持つべき全データ項目をリストアップする
3. **主キー（PK）の定義**：各エンティティを一意に識別するキーを定める（自然キー or サロゲートキー）
4. **多重度の整理**：エンティティ間の数量的な関係（1対多など）を定義する
5. **多対多の解消**：多対多の関係は中間テーブルを設けて解消する
6. **正規化の確認**：第3正規形を目安に、重複・依存関係を排除する
7. **レビュー**：開発者・業務担当者と一緒に確認し、担当者の確認でデータの実態と合っているかチェックする

## チェックリスト
- [ ] 全エンティティに主キー（PK）が定義されているか
- [ ] 全てのリレーションに多重度が定義されているか
- [ ] 多対多のリレーションが解消されているか（中間テーブルあり）
- [ ] 外部キー（FK）が正しく設定されているか
- [ ] 第3正規形を満たしているか（部分関数従属・推移的関数従属の排除）
- [ ] 全ての必須データ項目が属性として含まれているか
- [ ] 業務担当者と開発者双方のレビューが完了しているか
