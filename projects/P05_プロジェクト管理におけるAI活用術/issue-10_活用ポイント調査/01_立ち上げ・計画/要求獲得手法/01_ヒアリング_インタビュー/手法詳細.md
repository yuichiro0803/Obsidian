# ヒアリング / インタビュー：手法詳細

## 概要
ステークホルダーやエンドユーザーに対して直接質問を行い、ニーズや課題、暗黙知を引き出す手法。

## 目的と効果
- 要求のもとになっている背景や状況を引き出す
- 顕在化している要求を明らかにする
- 利害関係者のタイプを見極める

## 準備
- **事前知識の収集**:
  - **ドキュメント調査**: 既存の仕様書、マニュアル、過去の議事録、報告書の読み込み。
  - **組織・業務フローの確認**: 組織図や既存の業務フロー図を把握し、利害関係者の役割を予測する。
  - **既存システムの確認**: 現行システムの画面構成や、利用可能な場合は操作ログ・データ構造の確認。
  - **業界知識・トレンドのリサーチ**: 業界特有の用語、法規制、一般的なビジネスプロセス、最新技術動向の調査。
- **ヒアリング/インタビューの目的などの検討**:
  - **仮説の設定**: 事前知識をもとに、「おそらく課題はここにあるだろう」「この要件が重要になるはずだ」という仮説を立てる。
  - **ゴール定義**: このインタビューを通じて「何を明らかにしたいのか（What）」、「最終的にどんな状態にしたいのか（Outcome）」を明確にする。
  - **対象者の選定**: 仮説を検証し、ゴールを達成するために最も適したキーパーソンをリストアップする。
- **ヒアリング/インタビューの項目の検討・準備**:
  - **質問票（ヒアリングシート）の作成**: オープンクエスチョン（例：「どうお考えですか？」）とクローズドクエスチョン（例：「YesかNoか」）を適切に組み合わせた質問リストを作成する。
    - 🔗 [ヒアリングシート_テンプレート](./ヒアリングシート_テンプレート.md)
    - 🔗 [ヒアリングシート_フォーマット仕様書](./ヒアリングシート_フォーマット仕様書.md)
  - **ストーリーラインの構築**: 相手が答えやすい順序（現状の確認 → 課題の深掘り → 理想の姿）で質問を整理する。
  - **事前共有**: 必要に応じて、ヒアリングの目的と主な質問事項を事前に対象者へ送付し、回答の準備を促す。

## ヒアリング項目の枠組み
質問の抜け漏れを防ぎ、多角的に要求を引き出すための基本的なフレームワーク（観点）です。
- **現状業務の把握（As-Is）**:
  - 現在の業務フロー、利用しているシステムやツール、担当者の役割。
  - 発生している課題（ペインポイント）、不便に感じていること、ボトルネック。
- **理想の姿の定義（To-Be）**:
  - 課題が解決された状態、達成したいビジネス目標やKPI。
  - ユーザーが本当に望んでいる体験（理想のシナリオ）。
- **機能・非機能の要件候補**:
  - **機能的観点**: 「システムで何ができる必要があるか」「どんなデータが必要か」。
  - **非機能的観点**: 性能（レスポンス速度）、使い勝手（UI/UX）、セキュリティ、運用保守の制約。
- **制約事項・リスク**:
  - 予算、スケジュールに関わる制約。
  - 移行にあたっての障壁、ステークホルダー間の利害対立リスク。

## ヒアリングシートのサンプル
| カテゴリ | 質問項目（例） | 意図・目的 |
| :--- | :--- | :--- |
| **現状把握** | 現在の担当業務や役割について教えてください。 | 権限範囲や日常の関与度合いの確認 |
| | 現在の業務フローで最も時間がかかっている作業は何ですか？ | ボトルネック・ペインポイントの特定 |
| | その作業を行う際、どのようなシステムやツールを使っていますか？ | 技術的制約やシャドーITの確認 |
| **課題・理想** | 現状の課題が解決された場合、どのような状態になるのが理想ですか？ | To-Be（目標）の言語化 |
| | 最も優先順位高く解決したい課題を1つ挙げるとしたらどれですか？ | 要求の優先度づけ |
| **機能要求** | 新しいシステムに「必ず欲しい機能（Must）」は何ですか？ | コア機能の特定 |
| | 逆に、「あれば嬉しいが必須ではない機能（Want）」はありますか？ | スコープ調整のための分類 |
| **非機能要求** | システムが止まった場合、業務にどの程度影響が出ますか？ | 可用性・運用要件の確認 |
| | どのようなデータ（個人情報、機密情報など）を扱いますか？ | セキュリティ要件の確認 |
| **制約事項** | このプロジェクトを進める上で、懸念しているリスクはありますか？ | 潜在的なリスクの洗い出し |
| | 必須となるスケジュール（期限）や予算の制約は決まっていますか？ | プロジェクトマネジメント上の制約確認 |
