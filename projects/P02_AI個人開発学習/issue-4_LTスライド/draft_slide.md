# 【発表資料ドラフト】第1回：Antigravityで学ぶApex基礎

> 📋 所要時間：10分 ｜ 対象：Apex初学者、AIを活用してスキルアップしたいエンジニア

## スライド 0｜タイトル

```
コーディングの終焉と、設計の始まり

Antigravityで学ぶ Apex 基礎
〜 AIエージェントと共に歩む学習ロードマップ 〜

第1回：導入・基礎編
```

**話すポイント**
- 自己紹介（必要に応じて）
- このLTシリーズの概要（全3回）を一言で紹介

---

## スライド 1｜エンジニアの役割の変化（1分）

```
「書く」から「レビューする」へ

▶ 昔のエンジニア：コードをゼロから手で書く
▶ これからのエンジニア：AIが生成したコードの品質保証・レビュー

役割の変化 → "生産者" から "審査者・設計者" へ
```

**話すポイント**
- ChatGPTやCopilotの普及により、コーディングそのものは今後ますますAIが担う
- エンジニアに求められるのは「正しいコードを書けること」ではなく、「AIが生成したコードが正しいか判断できること」
- つまり、Apexを"理解する力"がより重要になる

---

## スライド 2｜Antigravity学習手順（2分）

```
AI（Antigravity）を使ったApex基礎学習の進め方

STEP 1：お題を決める
         → 過去の案件・業務で使えそなユースケースを1つ選ぶ
         → 迷ったらAIに「お題を提案して」と相談してOK
         → まずはユースケースを整理することを意識

STEP 2：AIにコードを生成させる
         → 「〇〇というケースで Apex コードを書いて」と指示

STEP 3：AIにコードを解説させる
         → 「このコードの各行を日本語で説明して」と依頼
         → その場で疑問が出たらすぐ聞ける
```

**話すポイント**
- 従来の学習：本やドキュメントを読む → 時間・理解コストが高い
- AIを使えば：お題を決めるだけで、生成→解説→深掘りまで完結できる
- ゼロから書く必要がないため、「理解」に集中できる

---

## スライド 3｜【実演①】コード生成〜解説（2分）

```
実演：お題を決めて、AIにコードを生成・解説させてみる

例）「商談（Opportunity）が成約したら、取引先責任者に
     自動でToDoを作成するトリガーを書いて」

▶ AIがコードを生成
▶ 「各行を日本語で説明して」と追加指示
```

> 💡 ライブデモ実施

**話すポイント**
- 実際に画面を見せながら操作する
- コードが出てきたら「なぜこう書く？」と聞いてみる
- 理解が深まる対話の例を見せる

---

## スライド 4｜【実演②】深掘り〜振り返りメモ作成（2分）

```
📌 発展編：理解を深め、メモを自動生成する

▶ 疑問点をAIに聞き、理解を深掘りする
   例）「ガバナ制限とは何か、このコードとどう関係するか説明して」

▶ やり取りをまとめてメモにしてもらう
   → 「これまでの質問内容を学習メモとしてまとめて」
   → 自分の苦手箇所を把握できる
   → 思考の癖を客観的に見られる
   → 振り返りに使える
```

> 💡 ライブデモ実施

**話すポイント**
- 「理解した気になる」だけでなく、AIとの対話内容が"振り返りメモ"になる
- ノートを自分で取る必要がなくなる（ドキュメント自動生成）
- 苦手箇所や思考パターンが可視化される → より効率的な学習サイクルへ

## スライド 5｜今後のロードマップ（参考）（2分）

```
Apex 学習の先にある未来

STEP 1 ✅ Apex 基礎（今回）
          1 つのクラスがかけるようになる

STEP 2 → 全体設計（次回以降）
          クラス構成を考える
          「このロジック、どのクラスに書くべき？」

STEP 3 → AIと伴走開発（将来)
          設計〜テストまで、AIと一緒に機能を完成させる
          要件定義 → 設計 → 実装 → テスト → コードレビュー
```

**話すポイント**
- 今回学ぶ「1クラス実装」は基礎の第一歩
- 実際の開発では、複数クラスが連携するアーキテクチャ設計が必要になる
- さらにその先には、요件定義からテストまでAIと伴走しながら開発するスタイルが待っている
- このシリーズで、段階的にその世界を体験できる
