# P03｜実行計画書 - 業務ヒアリング力の強化

> 本ドキュメントは、「このプロジェクトで何を達成するか」「誰が何にコミットするか」を明確にする実行計画書です。
> チームの羅針盤として、迷ったときは必ず本ドキュメントに立ち返ってください。

---

## 1. なぜこのプロジェクトをやるのか（Why）

### 現状の問題

```
業務ヒアリングは PLだけが行う → ボトルネックが生まれている

具体的な弊害：
  ・PLの属人化 → PLが不在だとヒアリングが進まない
  ・チームの業務理解が浅い → 「言われた通り作る」設計になりがち
  ・メンバーの心理的ハードル → 最終的にPBIの要件がどのような状態になれば「ヒアリング完了」と言えるのか、抽象的な要求事項を具体的な要件に落とし込むイメージが持てず、何を聞くべきか判断できない
  ・引き継ぎ・保守時に情報ギャップが発生する
```

### このプロジェクトで解決すること

> **「ヒアリングはPLだけの仕事」という状態を終わらせる**

---

## 2. 最終ゴール（What）

### ゴール文（1文で言うと）

> **チーム全員が、PLなしで業務ヒアリングを計画・実施・記録できる状態になること**

### ゴール状態の定義（達成の判断基準）

| # | ゴール | 確認方法 | 達成基準 |
|---|-------|---------|---------|
| G1 | メンバーがPBIを読んで「誰に・何を聞くか」を自分で設計できる | スプリントプランニングでの発言 | メンバーが自発的にヒアリング計画を立てる |
| G2 | PLなしでメンバーが単独ヒアリングを実施できる | 実施件数 | 月3件以上（チーム全体） |
| G3 | ヒアリング結果をPBIに構造的に記録できる | PBI記録率 | ヒアリング実施件数の80%以上に記録あり |
| G4 | 業務フロー図（Cacoo or Mermaid）をBacklog PBIに紐付けられる | PBI上のフロー図有無 | 新規PBIの50%以上にフロー図が添付される |
| G5 | チームでヒアリング内容を自発的に共有・議論している | 共有の自発性 | PLが促さなくてもSlack/レビューで共有が起きる |

## 3. スコープとスケジュール（When / Scope）

### 対象範囲

| 対象 | 内容 |
|------|------|
| **対象チーム** | 開発チームメンバー全員（PLを除く） |
| **対象業務** | Salesforce導入支援案件における新規PBIのヒアリング |
| **対象ツール** | Backlog（PBI管理）、Cacoo または Mermaid（フロー図）、PBIテンプレート |

### スコープ外（やらないこと）

- 要件定義書の完全作成（それはPLの役割）
- Salesforceの設計・実装レビュー
- 他チームへの展開（まず本チームで定着させる）

### フェーズ別スケジュール

```
Phase 0｜基盤整備（〜3月末）
  └─ 勉強会実施・PBIテンプレートの整備・Backlog連携手法の共有

Phase 1｜同行ヒアリング（4月 1〜2週）
  └─ PLが同行し、メンバーが主体でヒアリングを実施

Phase 2｜単独ヒアリング＋レビュー（4月 3週〜5月）
  └─ メンバーが単独でヒアリング。PLがPBIをレビュー

Phase 3｜自走（6月〜）
  └─ メンバーがヒアリング計画〜記録まで完結。PLはオンデマンドのみ
```

## 4. 実行計画（Who / Commitment）

### PLのコミットメント

| コミットメント | 具体的な行動 |
|-------------|------------|
| 勉強会を開催する | Phase 0内に1回（60〜90分）実施する |
| ヒアリングに同行する | Phase 1期間中、メンバーのヒアリングに同行しサポートする |
| PBIをレビューする | Phase 2期間中、週1回30分のPBIレビュー会を設ける |
| 関与を段階的に減らす | Phase移行タイミングを明示し、意図的に手を離す |

### メンバーへの期待（コミットメントのお願い）

| コミットメント | 具体的な行動 |
|-------------|------------|
| 勉強会に参加する | 参加率100%を目指す |
| 担当PBIで1件ヒアリングにトライする | 次スプリント以降、1人1件 |
| PBIテンプレートに記録する | ヒアリング当日中に記入する |
| 困ったことを共有する | 週次の振り返りで「詰まった点」を出す |

## 5. 成功指標の測り方（KPI）

| 指標 | 測定頻度 | 目標値 |
|------|---------|-------|
| ヒアリング実施件数（メンバー） | 月次 | 月3件以上 |
| PBI記録率 | 月次 | 80%以上 |
| Backlog PBIへのフロー図添付率 | 月次 | 新規PBIの50%以上 |
| 「業務理解不足による手戻り」件数 | スプリント毎 | 前スプリント比で減少傾向 |
