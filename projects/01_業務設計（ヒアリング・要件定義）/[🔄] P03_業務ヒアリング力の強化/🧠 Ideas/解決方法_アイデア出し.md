# 🧠 解決方法：アイデア出しドキュメント

## 1. 課題一覧 (Problem Statement)
| ID | 課題 (Challenge) | 具体的な内容 |
| :--- | :--- | :--- |
| **C1** | ヒアリングのバラツキ | 担当者の経験によって、深さや確認項目が大きく異なる |
| **C2** | 設計プロセスの属人化 | ユーザーストーリーから「何を聞くべきか」を導き出す方法が共有されていない |
| **C3** | 成果物の品質不均一 | ヒアリング結果を業務フロー等に落とし込む際の精度が一定ではない |

## 2. 課題別解決アプローチ (Solution Mapping)
| 課題 | 解決アプローチ (Approach) | 期待される効果 |
| :--- | :--- | :--- |
| **C1, C2** | **[💡B] 自動チェックリスト** | 全員が一定水準の質問項目を即座に準備できる状態にする |
| **C1, C3** | **[💡A] AIシミュレーター** | 実践前の模擬ヒアリングにより、若手のスキルを底上げする |
| **C2, C3** | **[💡C] 相互レビュー** | 他者の視点（レビュー）を入れ、設計の漏れやアウトプットの質を担保する |
| **C1, C3** | **[💡D] パターン・ライブラリ** | 典型的な業務（商談・請求等）の成功型を型化し、迷いをなくす |

## 3. 各アプローチの詳細 (Measures Detail)

### [💡A] AIヒアリング・シミュレーター
- **内容**: AIを顧客担当者に見立てて、ヒアリングの練習とフィードバックを行う。
- **メリット**: 失敗コストゼロで回数をこなせる。
- **実現難易度**: 低（プロンプトのみ）

### [💡B] ユーザーストーリー連動型チェックリスト自動生成
- **内容**: PBIを入力すると、Salesforce標準機能等を考慮した質問案を生成する。
- **メリット**: 準備時間の短縮と、決定的な聞き漏らしの防止。
- **実現難易度**: 中

### [💡C] 相互レビューの儀式化（ピア・ラーニング）
- **内容**: ヒアリング前の「設計シート」を、他メンバーが10分程度レビューする。
- **メリット**: 知識の共有が自然に行われ、アウトプットが安定する。
- **実現難易度**: 低（運用の工夫）

### [💡D] 成功事例のパターン・ライブラリ
- **内容**: 特定業務（例：会計連携）におけるヒアリング項目・フローの鉄板テンプレート。
- **メリット**: ゼロから考える必要がなくなり、スピードが向上する。
- **実現難易度**: 中（過去ナレッジの整理）

## 4. 優先順位とアクションプラン
1. **即効性重視**: [💡C] 相互レビューを次回のヒアリングから試験導入する。
2. **効率化重視**: [💡B] 自動チェックリストのプロンプトを試作し、有用性を検証する。
3. **長期育成**: [💡A] シミュレーターを用いた若手向け勉強会を企画する。
