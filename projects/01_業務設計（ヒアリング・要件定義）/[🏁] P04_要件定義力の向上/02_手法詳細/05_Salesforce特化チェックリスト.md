# 05｜要件定義チェックリスト（Salesforce特化）

## 概要
Salesforce上での実装を前提とした要件定義において、開発後半での手戻りを防ぐための確認項目。

## 1. 標準機能の最大限活用
- [ ] 標準の入力規則、自動化ツール（フロー）で実現可能か？
- [ ] 開発（Apex/LWC）が必要な場合、その正当な理由は何か？

## 2. プラットフォームのリミット考慮
- [ ] ガバナ制限（SOQLクエリ数、CPU時間等）を考慮したデータ処理か？
- [ ] API制限は問題ないか？（外部連携がある場合）

## 3. データ量とパフォーマンス
- [ ] 初期データ移行量、月間の発生データ量は？
- [ ] インデックスの活用が必要な検索・レポート要件か？（LDV考慮）

## 4. UI/UXとモビリティ
- [ ] Salesforceモバイルアプリ、タブレットでの操作性は考慮されているか？
- [ ] オフライン機能（利用場所の制約）の有無。

## 5. セキュリティと共有
- [ ] プロファイル、権限セット、共有設定で制御可能な要件か？
- [ ] 二要素認証（MFA）などのセキュリティポリシーとの整合性。
