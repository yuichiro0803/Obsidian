# 02｜メソッド・フレームワーク

## 1. 標準的な要件定義プロセス
1. **ビジネス目的の確認**: なぜこのプロジェクト・機能が必要か？
2. **ユーザーストーリーの抽出**: 「誰が」「何を」「なぜ」したいか？
3. **業務プロセスの可視化**: 現行（As-Is）とあるべき姿（To-Be）のフロー図。
4. **機能要件の定義**: 具体的な振る舞い、入力、出力、遷移。
5. **非機能要件の定義**: セキュリティ、パフォーマンス、保守性。

## 2. 推奨フレームワーク
- **INVEST原則**: 良いユーザーストーリーの基準。
- **MoSCoW法**: 優先順位付け（Must, Should, Could, Won't）。
- **ユースケース図・シーケンス図**: システム間のやり取りの明確化。

## 3. 要件定義のチェックリスト（Salesforce特化）
- [ ] 標準機能で実現可能か？
- [ ] ガバナ制限やAPIリミットを考慮しているか？
- [ ] データ量はどれくらいか？（ラージデータセット対応）
- [ ] モバイル対応、オフライン対応は必要か？
