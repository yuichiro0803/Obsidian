# WF05｜DFD（データフロー図）

## 概要
システム内でデータがどのように流れ、どこに蓄積・変換されるかを可視化する図。Salesforce導入案件では、データ連携要件の精査に特に有効。

## 主な構成要素
| 要素 | 記号 | 意味 |
| :--- | :--- | :--- |
| **外部エンティティ** | 四角 | システム外部のデータの発生源・受け取り先（例：外部システム、担当者） |
| **プロセス** | 円 / 角丸四角 | データを変換・処理する機能（例：申請処理、承認処理） |
| **データストア** | 二重線 | データが蓄積される場所（例：Salesforceオブジェクト、S3） |
| **データフロー** | 矢印 | データの流れと方向 |

## 活用シーン
- **データ連携案件**: 基幹システム、ERPとSalesforceのデータの流れを整理する際。
- **バッチ処理**: いつ・どのデータを・どこに連携するかを視覚的に確認する際。
- **スコープ確認**: Salesforceで管理するデータと、管理しないデータの境界を明確にする際。

## 作成ステップ
1. 外部エンティティ（連携元・先のシステムや担当者）を洗い出す。
2. 主要な処理（プロセス）を特定する。
3. 各プロセスで使うデータ（データストア）を特定する。
4. データの流れを矢印で結ぶ。
