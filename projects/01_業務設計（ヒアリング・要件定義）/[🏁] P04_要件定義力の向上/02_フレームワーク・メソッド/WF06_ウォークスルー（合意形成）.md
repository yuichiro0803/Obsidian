# WF06｜ウォークスルー（Walkthrough）/ 合意形成

## 概要
作成した仕様・設計ドキュメントをステークホルダーと一緒に「読み合わせる」プロセス。ウォーターフォールにおいては、工程の成果物の品質を保証するための重要な節目（マイルストーン）。

## 目的
- 定義した要件の「認識齟齬」を早期に発見する。
- ビジネスサイド（顧客）と技術サイド（開発）の共通理解を形成する。
- 仕様変更の負担が最も大きくなる「後工程での発見」を防ぐ。

## 実施タイミング（代表的な例）
| タイミング | 対象ドキュメント |
| :--- | :--- |
| 要件定義フェーズ末 | 機能要件一覧、業務フロー図（As-Is / To-Be） |
| 外部設計フェーズ末 | ユースケース記述、画面レイアウト |
| 内部設計フェーズ末 | クラス図、シーケンス図 |

## 進め方のポイント
1. **事前配布**: 資料は当日ではなく、数日前に配布して読んでくる時間をつくる。
2. **議事録の確定**: ウォークスルー中に出たコメント・承認事項は必ず議事録に残し、当日中に確定する。
3. **指摘と修正の管理**: 指摘内容は「軽微な修正」「要再ヒアリング」「スコープ外」に分類して管理する。
