# WF04｜ユースケース記述（テキスト形式）

## 概要
ユースケース図で示したアクターとシステムのやり取りを、テキストで詳細に記述する手法。特にウォーターフォールの「外部設計」フェーズで多用される。

## 基本フォーマット

**UC-001: 取引先を登録する**

| 項目 | 内容 |
| :--- | :--- |
| **アクター** | 営業担当 |
| **事前条件** | システムにログイン済みである |
| **事後条件** | 新規取引先レコードが保存されている |
| **基本フロー** | 1. アクターが「新規」ボタンを押す<br>2. 入力フォームが表示される<br>3. アクターが必須項目（名前、種別）を入力する<br>4. 「保存」ボタンを押す<br>5. システムが保存処理を行い、詳細画面へ遷移する |
| **代替フロー** | 3a. 必須項目が未入力の場合 → エラーメッセージを表示し、保存はしない |
| **例外フロー** | 5a. システムエラーが発生した場合 → エラーページを表示し、管理者に通知する |

## 使い分けのポイント
- **ユースケース図**: スコープの確認（What）に使う。
- **ユースケース記述**: 設計の詳細化（How）に使う。
- ウォーターフォールでは正式なドキュメントとして仕様書に含めることが多い。
