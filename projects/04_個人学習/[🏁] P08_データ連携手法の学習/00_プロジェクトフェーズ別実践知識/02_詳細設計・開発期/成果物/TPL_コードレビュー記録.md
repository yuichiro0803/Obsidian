## 重大度の定義

| レベル | 説明 | 対応 |
|:---:|:---|:---|
| 🔴 Critical | ガバナ制限超過・認証情報のハードコード・データ消失リスク | 必ず修正してから再レビュー |
| 🟠 Major | エラーハンドリング不備・テストカバレッジ不足 | 原則修正（例外は理由を記載） |
| 🟡 Minor | 命名規則の統一・コメントの追加 | 修正推奨（コメントでの説明も可） |

## チェックリスト

### ガバナ制限
- [ ] SOQL / DML がループ内に存在しないか
- [ ] リスト化してバルク処理しているか（収集→一括更新）

### セキュリティ
- [ ] URL・トークン・パスワードをソースコードにハードコードしていないか
- [ ] Named Credentials を使って認証情報を管理しているか
- [ ] 統合ユーザーの権限が最小権限になっているか

### エラーハンドリング
- [ ] try-catch ブロックが適切に設置されているか
- [ ] catch句が空（握りつぶし）になっていないか
- [ ] リトライ / Fatal のエラー分類が設計書に従っているか

### テスト
- [ ] テストカバレッジが 75% 以上か（目標: 85%）
- [ ] 異常系のテストケースがあるか（401 / 503 / CalloutException）
- [ ] `Test.setMock()` でHttpCalloutMockを設定しているか

### コーディング規約
- [ ] クラス・メソッドの命名規則が統一されているか
- [ ] 複雑なロジックにコメントが記載されているか
