# プロジェクトフェーズ別実践知識

本フォルダでは、データ連携プロジェクトにおける各フェーズ（要件定義〜保守運用）ごとの実務的なノウハウ、アンチパターン、チェックリストを整理します。
既存の技術習得フェーズ（Phase 1-3）を横断的に支える、PM/PL/SEとしての「動き方」に焦点を当てた知識ベースです。

## フォルダ構成

- **[[01_要件定義・基本設計期/|01_要件定義・基本設計期]]**
  - 非機能要件（セキュリティ・パフォーマンス）の合意形成
  - インタフェース一覧・基本設計書の定義方法
- **[[02_詳細設計・開発期/|02_詳細設計・開発期]]**
  - 詳細設計での共通部品化の判断基準
  - エラーハンドリング・リトライ処理の実装標準
- **[[03_テスト・移行期/|03_テスト・移行期]]**
  - 結合テストにおける外部システムシミュレーション
  - 本番移行時の切り戻し計画（ロールバック）の策定
- **[[04_保守・運用期/|04_保守・運用期]]**
  - 重大障害発生時のエスカレーションフロー
  - パフォーマンス劣化のモニタリング手法
