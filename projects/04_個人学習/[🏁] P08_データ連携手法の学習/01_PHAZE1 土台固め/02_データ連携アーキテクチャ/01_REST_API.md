# 01｜REST API アーキテクチャ

## 概要
REST (Representational State Transfer) アーキテクチャスタイルに基づく、軽量かつ柔軟なシステム連携手法。HTTPリクエスト（GET, POST, PUT, DELETEなど）を用いてリソースにアクセスし、CRUD操作を行う。

## 主な特徴
- **フォーマット**: 主に JSON を使用（可読性が高くデータサイズが小さい）。XMLにも対応。
- **通信方式**: 同期通信。リクエストに対して即座にレスポンスを返す。
- **ステートレス**: クライアントのセッション情報をサーバー側で保持しないため、拡張性が高い。
- **学習コスト**: シンプルな仕組みのため、開発者にとって学習ハードルが低く、導入が容易。

## ユースケース
- **モバイルアプリやWebアプリとの連携**: 軽量なデータ通信が求められる要件。
- **外部システムからSalesforceへの単一レコード作成・更新**: 例として、Webフォームからのリード登録や、他システムからのリアルタイムな顧客情報更新。
- **外部システムへの即時通知**: SalesforceのApexコールアウトから外部REST APIを呼び出して通知を行う。

## 設計および運用のポイント
- **APIリミットの考慮**: Salesforceにおける1日あたりのAPIコール数制限を消費するため、過度なポーリングは避ける。
- **認証**: OAuth 2.0 が標準的に利用される。
- **エラーハンドリング**: HTTPステータスコード（200 OK, 400 Bad Request, 500 Server Errorなど）に基づいた適切な再試行・エラー通知の設計が必要。
