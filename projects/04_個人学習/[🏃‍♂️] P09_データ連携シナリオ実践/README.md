# P09: データ連携シナリオ実践

Salesforceと外部システム間のデータ連携を、より実践的・具体的なユースケース（シナリオ）に沿って学習・検証するためのプロジェクトフォルダです。
基礎知識（P08）をベースに、要件定義から実装方針の検討までをシミュレーションします。

## 🎯 目的
- 実際の業務要件に近いユースケースを設定し、アーキテクチャの最適な選択と設計を練習する。
- 外部APIとの通信、エラーハンドリング、ガバナ制限回避のパターンプラクティスを蓄積する。

## 📝 進め方（学習サイクル）
シナリオを作成する際は、以下のステップでドキュメントを埋めていくことで、実務の設計プロセスをシミュレーションします。

1. **要件定義（シナリオ設定）**：データ量、リアルタイム性の要件、連携のトリガーを設定する
2. **方式検討**：複数ある連携手段（REST/Bulk/Streaming等）からメリット・デメリットを比較し、最適なものを選択する
3. **詳細設計・モックアップ**：実際に必要となるApexクラスの骨組みや、エラー設計・リトライ方針をまとめる
