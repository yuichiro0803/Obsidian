# SC02: Salesforceから基幹システムへの日次売上バッチ

## 1. シナリオ要件（ビジネス要件）
- **背景**: Salesforce上で営業が管理している「完了した商談（売上）」データを、毎晩1回、全社共通の基幹システム（ERP）へ同期させたい。基幹システム側で請求書発行等の処理を行うため。
- **データ量**: 1日あたり 約5,000〜10,000件
- **リアルタイム性**: 日次バッチ（夜間24:00〜明け方5:00の間に完了していればOK）
- **トリガー**: 日次（定期スケジュール起動）

---

## 2. 実装方式の検討

| 候補 | メリット | デメリット | 判定 |
|:---|:---|:---|:---:|
| **REST APIによる1件ずつのコールアウト** | 実装がシンプル。 | データ量が多い場合、ガバナ制限（Callout数, CPU時間制限）に抵触する確実性が高い。 | ✕ |
| **Batch Apex + REST API Bulk送信** | Salesforce標準機能で定期実行可能。リストに詰めてバルク送信すれば制限を回避可能。 | 外部システム側が「リストの一括受け取りJSONエンドポイント」に対応している必要がある。 | ◯ |
| **Salesforce Bulk API V2**<br>（基幹システム側主導） | 大量データの抽出に最適（SF側に負荷をかけない）。 | 基幹システム（またはETL等のミドルウェア）側からSFへ定時ログインしてデータを取りに来る処理が必要。 | ◎ |
