# 02｜技術要素・パターン

## 1. 連携プロトコル・形式
- **REST API**: JSONベース。デファクトスタンダード。
- **SOAP API**: XMLベース。レガシーシステムや厳密な型定義が必要な場合。
- **GraphQL**: 必要なデータだけを取得。オーバーフェッチの防止。
- **gRPC**: 高速なバイナリ通信。マイクロサービス間。

## 2. 連携パターン
| パターン名 | 特徴 | 主な用途 |
| :--- | :--- | :--- |
| **Request-Response** | 即時に結果が必要 | 在庫確認、与信照会 |
| **Fire-and-Forget** | 送信して完了（非同期） | ログ送信、通知 |
| **Batch Sync** | まとめて処理 | 前日の売上データ同期 |
| **CDC (Change Data Capture)** | 変更検知して即同期 | リアルタイムDB同期 |

## 3. 認証・認可
- **OAuth 2.0**: JWTベアラートークンフローなど。
- **API Key**: シンプルな認証。
- **Mutual TLS (mTLS)**: 証明書による双方向認証。

## 4. エラーハンドリング・リトライ
- 指数関数的バックオフ（Exponential Backoff）
- べき等性（Idempotency）の確保
- デッドレターキュー（DLQ）の活用
