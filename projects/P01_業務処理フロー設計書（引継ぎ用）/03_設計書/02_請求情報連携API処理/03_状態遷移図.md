# 03｜状態遷移図

> [🔙 INDEX](./00_INDEX.md)

## 1. 状態遷移定義

- **対象オブジェクト**: 請求先（`BillingInformation__c`）および請求書情報（`InvoiceHeader__c`）
- **対象ステータス項目**: 連携ステータス（`SendStatus__c` / `Status__c`）

> **備考**: ERP連携処理に伴うステータスの変遷を定義します。対象データがSalesforce側の処理で「確定済（未連携）」となっている前提で記載しています。

```mermaid
stateDiagram-v2
    direction LR

    state "確定済(未連携)" as 0
    state "連携中" as 1
    state "連携済" as 2
    state "連携エラー" as 3
    
    [*] --> 0 : Salesforce側で確定 (Status=0)
    
    0 --> 1 : [E-01] データ抽出API (GET) (Status=1)
    
    1 --> 2 : [E-02] 結果受信 (成功:1) (Status=2)
    1 --> 3 : [E-02] 結果受信 (失敗:2) (Status=3)
    
    3 --> 0 : エラー解消後の手動リトライ
    
    state "SmartDB承認プロセス" as SDB {
        direction TB
        state "未定" as undecided
        state "承認済" as approved
        state "却下" as rejected
        
        [*] --> undecided
        undecided --> approved : [E-03] Webhook (ERP連携完了等)
        undecided --> rejected : [E-03] Webhook (取消/再申請等)
    }
    
    2 --> SDB : [E-03] Webhook受信
    rejected --> 0 : 再作成の検討
```
