# 03｜状態遷移図

> [🔙 INDEX](./00_INDEX.md)

## 1. 状態遷移定義

- **対象オブジェクト**: 請求先（`BillingInformation__c`）および請求書情報（`InvoiceHeader__c`）
- **対象ステータス項目**: 連携ステータス（`SendStatus__c` / `Status__c`）

> **備考**: ERP連携処理に伴うステータスの変遷を定義します。対象データがSalesforce側の処理で「確定済（未連携）」となっている前提で記載しています。

```mermaid
stateDiagram-v2
    direction LR

    [*] --> 確定済_未連携 : Salesforce側の操作で確定
    
    確定済_未連携 --> 連携中 : [E-01] データ抽出API実行 (GET)
    
    連携中 --> 連携済 : [E-02] 結果更新API実行 (成功時)
    連携中 --> 連携エラー : [E-02] 結果更新API実行 (失敗時)
    
    連携エラー --> 確定済_未連携 : エラー解消後の手動リトライ等
    
    連携済 --> [*] : [E-03] 承認状況・SmartDBステータスの更新等
```
