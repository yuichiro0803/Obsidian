# 📊 P01｜業務理解（業務処理フロー設計書）

🔗 [GitHub Project #7](https://github.com/users/yuichiro0803/projects/7) ／ 📋 [実行計画書・ゴール定義](./実行計画書_ゴール定義.md)

## 概要
Salesforce 導入時などに発生する「業務フロー図だけでは表しきれないシステム内部の動作（自動処理・状態遷移・エラーなど）」を一括管理するための設計フォーマットと、その作成ガイド群をまとめたプロジェクトです。

---

## 📁 フォルダ構成と使い方

目的・用途に応じてフォルダを明確に分離しています。あなたが「今やりたいこと」に合わせて該当フォルダを開いてください。

### 1. 📖 設計書の作り方を知りたい
**👉 `01_設計手順・ガイド/`**
- 設計手順や、特有の記述ルール（状態遷移図の書き方など）をまとめた**教科書**です。
- [【必読】業務処理フロー設計書の作成手順.md](./01_設計手順・ガイド/【必読】業務処理フロー設計書の作成手順.md) から読み始めてください。

### 2. 📝 実際に設計書を書きたい（実作業）
**👉 `02_テンプレート・サンプル_実作業用/`**
- コピーしてそのまま使える**テンプレート**と、書き方の参考となる**完成済みサンプル**が格納されています。
- 実際に業務設計を行う際の手元作業用フォルダです。

### 3. 🏗️ 作成した設計書を管理・参照したい
**👉 `03_設計書/`**
- 本プロジェクトのフォーマットに沿って作成された、各業務の**設計書本体**を格納するフォルダです。
- 過去の設計を参照したり、最新の仕様を確認したりするためのディレクトリです。

### 4. 📚 設計のテクニック・パターンを知りたい
**👉 `04_参考資料/`**
- 特定の記述方法やパターンなど、設計を深掘りするための**参考資料**です。
- [【ガイド】Salesforce状態遷移図記述パターン集.md](./04_参考資料/【ガイド】Salesforce状態遷移図記述パターン集.md) などが格納されています。

### 5. 🤔 そもそもなぜこのフォーマットになったか背景を知りたい
**👉 `99_検討経緯・参考情報/`**
- プロジェクト立ち上げ当時の課題意識や、解決策に至るまでの検討プロセス（メモや論点整理）が保管されています。
- 実作業で必須ではありませんが、「なぜこういう項目があるの？」と疑問に思った際の**参考資料・アーカイブ**として機能します。
