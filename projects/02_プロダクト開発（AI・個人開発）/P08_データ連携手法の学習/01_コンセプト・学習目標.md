# 01｜コンセプト・学習目標

## 1. ビジョン
「点と点を結び、データが淀みなく流れるシステムアーキテクチャを設計・実装できるエンジニアになる」

## 2. 学習の背景
- Salesforce案件において外部システム（ERP, EC, DWH等）との連携は不可避。
- リアルタイム連携かバッチ連携かの判断基準を明確にしたい。
- エラーハンドリング（再試行ロジック、不整合検知）の堅牢なパターンを身につけたい。

## 3. 学習目標
- **基礎**: HTTPメソッド、認証（OAuth 2.0）、JSON/XMLの高度な扱い。
- **パターン**: リクエスト-リプライ、Fire-and-Forget、パブリッシュ-サブスクライブ。
- **実践**: 
  - Salesforce 公式 API (REST/Composite) の使いこなし。
  - Webhook を受け取るバックエンド (Node.js/GAS) の構築。
  - 大量データ（Bulk API）の効率的な同期手法。

## 4. 最終的なアウトプット
- 典型的な連携シナリオ（マスタ同期、トランザクション連携）の設計テンプレート作成。
- 自作の検証用モックサーバー環境。
